== Non-Goals

Things bundeno will NOT do:

* **Node.js compatibility** — Use Deno/Bun's built-in Node compat
* **Polyfills** — We delegate to native APIs, not fake them
* **Build tools** — Use Deno/Bun's native tooling
* **Browser support** — Focus is server-side runtimes

== Future Possibilities

Ideas for post-1.0:

* Edge runtime support (Cloudflare Workers, Deno Deploy)
* WASM fallbacks for unsupported platforms
* Plugin system for custom compat modules
* Performance profiling tools
